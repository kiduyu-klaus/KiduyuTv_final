---

Organize screen components into platform-specific packages and refactor navigation to support separate TV and mobile UI flows.

Key changes:

  • Package Reorganization:

- Relocated TV-specific screens (Home, Movies, TvShows, Search, Settings, Detail, Cast) into new .tv subpackages.

- Relocated mobile-specific screens and components into new .mobile subpackages.

- Moved mobile UI components like MobileBottomNavigation, MobileMovieCard, and MobileMediaCards to ui.components.mobile.

  • Navigation Updates:

- Updated NavGraph.kt and MobileNavGraph.kt to use the new package structures for both TV and mobile destinations.

- Refactored MobileNavGraph to improve code readability by standardizing route creation and parameter encoding for movie/TV details and cast views.

- Integrated PlayerActivity and mobile-specific detail screens directly into the mobile navigation flow.

  • Code Refactoring & Cleanup:

- Standardized imports across multiple files, replacing fully qualified names (e.g., androidx.compose.ui.platform.LocalContext) with simple imports.

- Fixed missing imports for Context, URLEncoder, and BorderStroke in mobile screen implementations.

- Updated MobileSettingsScreen to use the BuildConfig import for version naming.

- Cleaned up unused imports and redundant modifiers (like focusable and graphicsLayer) in several UI components.

  • View Model Integration:

- Updated StreamLinksViewModel and MobileCastDetailViewModel to reflect changes in screen locations and state management.

v1.1.88 Update stream provider configurations and refine the WebView-based video player to improve playback tracking and stability. — v1.1.88 Stable April 12, 2026 ↗ View on GitHub

Branch: main

---

Update stream provider configurations and refine the WebView-based video player to improve playback tracking and stability.

Key changes:

  • Stream Provider Updates:

- Updated the VidSrc provider URL to a new endpoint (vidsrc-embed.ru) and added autoplay/autonext parameters for both movies and TV shows.

  • Player Progress Tracking:

- Refactored the injected JavaScript in PlayerActivity to use a MutationObserver for more reliable detection of video elements.

- Enhanced the JavaScript-to-Native bridge to better extract content metadata (ID, type, season, episode) directly from the URL.

- Implemented enforceVolume logic in the player to ensure video is unmuted and set to maximum volume.

- Improved the reliability of the 15-second synchronization loop by leveraging cached data from the JavaScript interface.

  • Code Quality & Formatting:

- Standardized code formatting and logging alignment across PlayerActivity.

- Simplified several conditional expressions for determining season and episode information during progress saves.

- Cleaned up redundant JavaScript calls and optimized the ad-blocking and redirect-blocking scripts.

  • Bug Fixes:

- Fixed an issue where the VidKing provider detection logic was missing specific URL patterns.

- Ensured consistent saving of final playback positions for both movies and TV shows when the activity is destroyed.
